Data Interpretation Strategy:
Scan vs. Read

0:00-0:20 | Hook: Data Interpretation (DI) questions are time-consuming if approached wrong. The secret is to scan first for structure, then read carefully only what is needed.

Phase 1

The 30-second scan

Before reading any questions, spend 30-45 seconds scanning the data display.

Element Why It Matters
Title What is this data about?
Axis labels What is being measured?
Units Thousands? Millions? Percent?
Time period What years or range does it cover?
Scale Does the y-axis start at zero?
0:20-2:00 | The scan phase: 30-45 seconds before touching any question. Check the title, axis labels, units, time period, and scale. This prevents unit errors and misreading the data source.

Phase 2

Read the question precisely

Key words change the entire meaning of a question.

Key Word Meaning
"Approximately" Estimation is okay; round freely
"Exactly" Be precise; no rounding
"At least" / "At most" Minimum ($\geq$) / Maximum ($\leq$)
"From X to Y" Change between two specific points
"During X to Y" Total over an entire period
2:00-3:30 | The read phase: look for key words that change meaning. "Approximately" means you can round. "At least" means greater than or equal to. "From X to Y" asks about change between two points. "During X to Y" asks about the total over a period.

Part 2

Question Prioritization

Not all Data Interpretation questions are equal

3:30-3:40 | Transition: Now that you know how to scan and read, let's talk about which questions to answer first.

Strategy

The two-pass approach

Pass 1

Easy Questions First

  • Direct reads from one graph
  • Simple comparisons
  • Single data source questions
Pass 2

Return to Hard Ones

  • Multi-step calculations
  • Questions requiring both displays
  • Complex percent changes

EASY Q1: What were East sales in 2023?

MEDIUM Q2: % increase from East to West?

HARD Q3: Which region grew most from '22-'23?

3:40-5:00 | The two-pass strategy. Pass 1: answer easy questions first (direct reads, simple comparisons). Pass 2: return to harder multi-step questions. This banks easy points and builds familiarity with the data.

Time Budget

Triage by difficulty

Difficulty Example Time
Easy Direct read from one graph 30-45 sec
Medium Calculate from extracted values 60-90 sec
Hard Multi-step, multiple sources 2-3 min
Skip Signal

If you have spent 2+ minutes without progress, mark it and move on. Return with fresh eyes after completing others.

5:00-5:45 | Time budgets: easy questions under 45 seconds, medium around 90 seconds, hard up to 3 minutes. If stuck for 2+ minutes, skip and return. Do not let one question steal time from the rest.

Avoid These

Four efficiency traps

Trap 1

Reading Everything First

Let questions guide your reading.

Trap 2

Solving in Order

Triage by difficulty instead.

Trap 3

Over-Precision

Check answer spacing before calculating.

Trap 4

Misreading Units

Circle or underline units during scan.

5:45-6:30 | Four efficiency traps. Trap 1: do not read everything before looking at questions. Trap 2: do not solve in order, triage instead. Trap 3: do not over-calculate when answers are spread apart. Trap 4: circle units during your scan to prevent misreading.

Key Takeaways

DI strategy: Scan vs. read

  • Scan first (30-45 sec) -- Title, labels, units, scale, and legend before touching questions
  • Read precisely -- Key words like "approximately," "at least," and "from X to Y" change everything
  • Two-pass strategy -- Answer easy questions first, return to hard ones second
  • Budget your time -- Easy: 30-45 sec, medium: 60-90 sec, hard: 2-3 min
  • Skip and return -- If stuck for 2+ minutes, mark it and move on
  • Let questions guide you -- Do not read everything up front; let each question tell you where to look
6:30-7:00 | Summary: Scan for structure, read for precision, solve efficiently. Use the two-pass strategy. Budget time by difficulty. Skip and return when stuck. Let questions guide your reading.
